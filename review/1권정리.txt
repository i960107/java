논리를 표현하기 위해 사용
- boolean: true/false. 1byte
3. 형 변환이란 무엇이고 왜 해야 하나요?
타입을 변환하는 것을 말하며 자바의 다형성을 활용하기 위해 혹은 기본 타입 변수들의 연산시 같은 타입으로 맞춰주기 위해 사용된다.
형변환에는 묵시적 형변환과 명시적 형변환이 있는데, 묵시적 형변환은 기본 타입의 경우 더 큰 크기의 타입으로 변환하는 것, 참조 타입의 경우 부모 타입으로 변환하는 것을 말하며
jvm에 의해 자동으로 실행된다. 명시적 형변환은 기본 타입의 경우 더 작은 타입으로 변환하는 것, 참조 타입의 경우 자식 타입으로 변환하는 것을 말하며 이때 (변환할 타입)변환할 대상
으로 명시해주어야한다. 기본타입은 메모리를 벗어나는 값을 버리기 때문에 값이 달라질 수 있고, 참조 타입의 경우 object type이 변활할 타입과 다르다면
Class Casting Exception이 발생할 수 있다.
4. if 문의 용도는 무엇이며, if-else와 if-else if는 어떤 점이 다른지 정리해주세요.
if문은 조건에 따라서 실행흐름을 분기하기 위해서 사용된다. if는 조건이 참일 경우 거짓일 경우 두가지로 나눠서 처리하며, if-else if는 여러 경우로 나누기 위해서 사용된다.
첫번째 조건문부터 내려오면서 처음으로 조건을 만족하는 경우에 블록이 실행되므로 작은 범위의 조건부터 명시해주는 것이 좋다.

13. 인터페이스, abstract 클래스, enum클래스가 있는데 각각의 특징 및 다른 점을 정리해주세요.
인터페이스와 abstract 클래스는 클래스 설계시에 함수를 선언만 하고 구현은 실제 객체들에 위임한다. 구현을 숨기고 선언만 드러내어 외부와 상호작용하는 방법을 정해둔다.
인터페이스는 모든 메서드가 구현을 포함하면 안되고, abstract 클래스는 abtract 메서드를 0개 이상 포함하며 abstract 메서드는 구현하면 안된다.
인터페이스의 함수 선언은 public abstract를 암시하며 abstract 클래스의 함수는 private 이면 안된다.
implements interface, extends 클래스로 상속할 수 있으며 무조건 구현해야한다.
enum클래스는 관련있는 enum constant들의 집합을 관리하기 위해 사용되며, 상수보다 좀 더 복잡한 로직을 수행할 수 있다. java.lang.Enum을 상속받게 되며
각 enum 객체들을 singleton으로 힙 메모리에 생성된다.

20. try-catch-finally 블록은 왜 사용하고 각각의 블록이 어떤 용도로 사용되는지 정리해 주세요.
예외를 처리하기 위해 사용된다. 예외가 발생하면 스레드가 종료되지만 예외를 처리해주면 스레드가 계속 실행될 수 있다.
try블록 실행중에 예외가 발생하면 그 이후 문장을 실행하지 않고 catch로 넘어간다. 만약 catch 하는 예외 중 해당하는 예외가 있으면
catch블럭이 실행된다. finally는 예외가 발생하든 발생하지 않든 실행된다.
