1. 패키지를 선언할때 사용하는 예약어는? package
2. 패키지 선언은 클래스 소스 중 어디에 위치해야 하나요? 최상단. 어떤 자바 코드도 앞에 오면 안된다.
3. 패키지를 선언할때 가장 상위패키지의 이름으로 절대 사용하면 안되는 단어는? java
4. 패키지 이름에 예약어가 포함되어도 되나요? no
5. import는 클래스 내에 선언해도 되나요? 아니요. package 선언 후 class 시작 전.
6. 같은 package에 있는 클래스를 사용할때 import 해야하나요? 아니요. 다른 패키지의 경우만 파일 경로를 명시해주기 위해서 사용.
7. 특정 패키지에 있는 클래스들을 모두 import할때 사용하는 기호는 무엇인가요? *
8. 클래스에 선언되어 있는 static한 메소드나 변수를 import 하려면 어떻게 선언해야 하나요?  import static
9. 접근 제어자 중 가장 권한이 넓은 어떤 클래스에서도 접근할 수 있는 것은 무엇인가요? public
10. 접근 제어자 중 가장 권한이 좁은 다른 클래스에서 접근할 수 없는 것은 무엇인가요? private
11. 접근 제어자 중 같은 패키지와 상속 관계에 있는 클래스만 접근할 수 있도록 제한하는 것은 무엇인가요? protected
12. Calculate.java라는 소스가 있을 경우, 그 소스 내에는 Calculate 클래스 외에는 ()으로 선언된 클래스가 있으면 안된다. public
[참고] package가 없으면? -> 컴파일 에러 발생. 컴파일은 루트 디렉토리에서 이루어져야하며 지정한 package명으로 해당 클래스를 찾는다.
