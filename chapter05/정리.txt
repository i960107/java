1. 값을 할당할 때 사용하는 연산자의 기호는 무엇인가요? = 대입 연산자.
2. 기본적인 덧셈, 뺄셈, 곱셈, 나눗셈, 나머지를 계산할 때 사용하는 연산자의 기호는 무엇인가요? + - * / %
3. += 는 무엇을 할 때 사용하는 연산자인가요? 복잡 연산자로 왼쪽 항의 원래 값에 오른쪽 항의 값을 더한 결과를 왼쪽 항에 대입하라는 연산
4. 여러분들이 연산의 순서를 모르거나 확실히 하고 싶을때 어떤 기호를 사용해야하나요? () 가장 우선순위 높음
5. ==와 !=의 차이는 무엇인가요? 동등성 비교에 사용되는 연산자로 ==은 같다면 true 다르다면 false. != 은 다르다면 true 같다면 false가 된다.
6. <와 <=의 차이는 무엇인가요? 등호의 차이. <은 a가 b보다 작을때 true 그렇지 않으면(b가 a와 같거나 a보다 클때)
<=은 a가 b보다 작거나 같을때 true, 그렇지 않으면 false가됨.
7. !연산자는 어떤 타입에 사용할 수 있나요? boolean 타입에만 사용가능
8. ?:로 표시하는 삼항 연산자와 ? :뒤에 명시에주는 값은 무엇을 의미하나요 대입할 변수 = (조건식) ? (조건식이 true일때 결과) : (조건식이 false일때 결과)
9. 자바는 형 변환을 한다고 했는데 short의 값을 long에 할당할때에는 어떤 것을 해주어야 하나요? 둘다 숫자형이기 때문에 casting이 가능하고,
 short는 2바이트, long은 4바이트이기 때문에 upcasting에 해당한다. upcasting은 자동으로 이루어지기 때문에 어떤 것도 해주지 않아도 된다.
10. 반대로 Long 값을 short에 할당할때는 어떤 것을 해주어야하나요? downcasting은 명시적으로 형변환을 해주지 않으면 컴파일 오류가 생기기 때문에,
(short)를 변수명앞에 써주어야한다.
11. 9와 10의 경우 중 어떤 경우가 기존 값이 사라지고, 엉뚱한 값으로 바뀔 수 있나요? 10과 같은 downcasting의 경우 byte가 줄어드는 것으로 앞쪽부터 줄어든 크기의 byte를 버린다.
 따라서 short범위를 벗어난 long값을 downcasting할 경우 엉뚱한 값이 될 수 있다.
