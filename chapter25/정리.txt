1. 스레드와 프로세스의 차이를 이야기해보세요. 프로그램이 시작되면 운영체제로부터 메모리를 할당받아서 하나의 프로세스가 실행된다. 스레드는 프로세스 내 실행흐름으로
여러개 있을 수 있으며 프로세스의 스택 메모리 일부를 할당받는다. 프로세스 내에서 여러개의 작업을 동시에 수행하려고 할 때 스레드를 사용할 수 있다.
2. 여러분들이 쓰레드 클래스를 만들기 위해서는 어떤 인터페이스를 구현하면 될까요? Thread클래스를 확장하거나 Runnable인터페이스를 구현할 수 있다.
자바는 다중상속을 지원하지 않기 때문에 다른 클래스를 상속받아야할때 Runnable인터페이스를 사용하고, 그렇지 않으면 Thread를 사용하는 것이 좋다. Runnable 인터페이스를
구현하여도 실제 스레드를 실행시키기 위해서는 Thread객체를 생성해야 하기 때문이다.
3. 2에서 이야기한 인터페이스에 선언되어있는 유일한 메소드는 무엇인가요? public void run()
4. 쓰레드가 시작되는 메서드 이름은 무엇인가요? run()
5. 쓰레드를 시작하는 메서드의 이름은 무엇인가요? start(). 직접 구현하는 것은 run()이지만 start()를 통해서만 직접 실행할 수 있다. jvm이 스레드를 관리하는
작업이 추가된다.
6. 쓰레드에 선언되어있는 sleep() 메소드의 역할은 무엇인가요? static함수로 선언되어있으며 현재 실행중인 쓰레드를 잠시 중지(대기)한다.
7. sleep() 메소드를 사용할때에는 try-catch()로 감싸주어 예외를 처리해주어야 하는데, 그 이유는 무엇인가요?
join() 혹은 sleep()시에는 InterupptedException이 발생할 수 있는데 컴파일 에러이기 때문에 처리가 강제된다. 잠시 중지한 WAITING, TIMED_WAITING
상태에서 interrupt()함수가 실행되면 InterruptedException을 발생시키면서 쓰레드가 종료된다.
8. 데몬 쓰레드와 일반 쓰레드의 차이는 무엇인가요? 데몬 쓰레드는 부가적인 쓰레드로 종료되지 않아도 메인 쓰레드가 중지되고 프로그램이이 종료될 수 있다. 일반
쓰레드는 하나라도 실행이 완료되지 않아서 중지되지 않은 상태라면 (RUNNABLE, WAITING, TIMED_WAITING) 메인 쓰레드가 종료되지 않는다.
9. synchronized 구문은 왜 써주며, 어디에 사용해야 하나요? 공유하는 자원에 여러 메서드가 동시에 접근시 race condition, deadlock과 같은 동시성 문제가
발생할 수 있기 때문에 해당 자원은 한번에 하나의 쓰레드만 접근할 수 있도록 할때 사용한다.
10. synchronized를 사용하는 두가지 방법은 어떤 것인가요?
1) synchronized method 메서드에 선언
2) synchronized block: 잠금객체를 사용해서 메서드 내 공유 변수에 접근하는 특정 부분만 동기화 선언
11. 쓰레드의 상태에는 어떤 것들이 있나요?
- NEW: 쓰레드 객체가 생성되었지만 실행되지 않은 상태.
- RUNNABLE: start()가 호출되어 실행 중인 상태.
- BLOCKED: 쓰레드가 실행 중지 상태이며, 모니터락이 풀리기를 기다리는 상태.
- WAITING: sleep(), join()등으로 대기중인 상태.
- TIME_WAITING: 일정 시간 동안 대기중인상태.
- TERMINATED: 쓰레드의 run()이 모두 실행되었거나 interrupt()가 호출되어 종료된상태.
12. 쓰레드에 선언되어있는 join()메소드의 용도는? 현재 쓰레드에서 해당 쓰레드가 종료되기를 기다린다는 의미.
13. 쓰레드에 선언되어있는 interrupt() 메서드의 용도는 무엇인가요? 쓰레드가 WAITING 상태일때 interrup()를 호출하면 InterruptedException을
발생시키고 쓰레드가 종료됨.
14. interrupt()메서드를 호출하면 해당 쓰레드는 어떤 상태에 있을래 interrupt() 효과가 발생되나요? WAITING 상태일때. 그렇지 않으면 계속 수행됨.
15. Object 클래스에 선언된 wait() 메소드의 용도는 무엇인가요? 해당 객체의 모니터에 대기하기 위해서 현재 스레드를 대기시킨다.
16. Object 클래스레 선언된 notify() 메소드의 용도는 무엇인가요? 해당 객체의 모니터에 대기하고 있는 쓰레드 중 가장 오래된 객체 하나를 깨운다.
18. ThreadGroup에 선언된 enumerate()메서드의 용도는 무엇인가요? 매개변수로 넘긴 쓰레드가 속한 쓰레드 그룹의 쓰레드들을 배열에 담는다.
